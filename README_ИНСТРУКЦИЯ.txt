# Что внутри архива

- **index.html** — чистый лёгкий HTML (63 КБ вместо 2,5 МБ)
- **images/** — папка с 6 картинками, которые раньше сидели внутри HTML:
  - image_01.jpg, image_02.jpg и т.д.

---

# Что сделать на GitHub (пошагово)

## Шаг 1. Распакуй архив

Распакуй `landing_clean.zip` на рабочий стол. Получится папка с `index.html` и `images/` внутри.

## Шаг 2. Зайди в свой репозиторий GitHub

Открой репозиторий, в котором лежит старый лендинг (тот, что подключён к Vercel).

## Шаг 3. Удали старый index.html

- Кликни на файл `index.html` в списке файлов
- Нажми значок **мусорной корзины** (справа вверху, рядом с кнопкой Raw)
- Внизу страницы нажми зелёную кнопку **Commit changes**

## Шаг 4. Загрузи новые файлы

- Вернись в корень репозитория (нажми название репо вверху)
- Нажми кнопку **Add file** → **Upload files**
- Перетащи в окно загрузки:
  - новый `index.html`
  - **папку `images` целиком**
- Внизу страницы — зелёная кнопка **Commit changes**

## Шаг 5. Проверь сайт

Через 30–60 секунд Vercel автоматически обновит сайт. Открой свой адрес — всё должно выглядеть как раньше, но страница будет грузиться в 3–5 раз быстрее.

---

# На будущее (важно!)

**Никогда больше не встраивай картинки прямо в HTML.** Всегда клади их в папку `images/` и ссылайся через `src="images/фото.jpg"`. Тогда HTML останется лёгким, его можно будет редактировать в любом чате Claude, открывать в GitHub, и сайт будет грузиться быстро.

Если понадобится переделать этот лендинг — теперь ты сможешь спокойно вставить `index.html` (63 КБ) в новый чат и попросить любые правки.
